01 Background

TeraCyte builds AI-powered single-cell analysis tools for precision medicine. One core task is classifying individual cells as live or dead directly from brightfield (BF) microscopy images — without relying on fluorescent stains.

BF-based live/dead classification is challenging because:

  • The signal is subtle (membrane integrity changes are small in BF)
  • Microscopy data has significant batch effects across imaging sessions
  • Real datasets are messy: out-of-focus acquisitions, empty wells, class imbalance

This assignment asks you to build a binary live/dead classifier and, just as importantly, to demonstrate that you understand the data before you model it.

03 Task

  1. Explore the dataset. Report basic statistics: class balance, per-experiment distributions, image quality observations. This section is worth as much as the model.
  2. Design and justify a train / validation / test split. Submit a CSV file (split.csv) with columns sample_id, split where split ∈ {train, val, test}. Your justification must address data leakage.
  3. Train a binary live/dead classifier (any DL framework, any architecture). The input is the 224×224 BF PNG; the output is live or dead with a confidence score.
  4. Evaluate and analyse your model. Report per-experiment metrics. Discuss the failure modes you observe and how you would address them in a production system.

05 Report

Your analysis.ipynb notebook (executed, all outputs visible) should cover:

  1. Data exploration — class balance, per-experiment distributions, image quality observations
  2. Split design rationale — why your split is valid and what leakage risks you avoided
  3. Model choice and training — architecture, loss function, augmentations, why you chose them
  4. Training monitoring — loss and metric curves, learning rate schedule, convergence behaviour
  5. Performance metrics — confusion matrix, per-experiment accuracy / F1, precision-recall curve
  6. Error analysis — at least 5 misclassified examples with commentary; what patterns do you see?
  7. What you would do with more time — top 3 improvements

07 Practical Notes

  • You may use any framework (PyTorch, JAX, etc.) and any pretrained backbone.
  • Training on a free GPU (Colab, Kaggle, RunPod) is perfectly fine; note the GPU model in your README.
  • The dataset is ~89 MB uncompressed (~69 MB as a zip). Metadata alone is ~2 MB.

We care more about how you reach your results than about the final accuracy number. A clean, well-reasoned process with honest failure analysis is more valuable than a squeezed leaderboard score.
